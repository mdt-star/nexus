# 项目架构设计

## 一、项目概述

`mdt-star/nexus` 是一个 Laravel 核心权限模块，提供功能权限控制、菜单管理、数据范围策略、动态配置等基础设施能力。

---

## 二、核心概念

### 两个核心权限概念

| 概念 | 说明 | 控制层面 | 生效范围 |
|------|------|---------|---------|
| **Permission** | 功能权限标记（Feature Flag），通过 tag 标识 | 路由/页面/元素 | 前后端请求层 |
| **ModelAccess** | 模型访问权限，控制主体对特定模型的读写删及数据范围 | Eloquent Model | 模型层（ORM） |

> **Permission** 控制"能不能看到/访问这个按钮或页面"；**ModelAccess** 控制"能不能查/改/删这类数据"。

### 其他概念

| 概念 | 说明 |
|------|------|
| Menu | 菜单发布池，前端可用的菜单项 |
| Desktop | 用户桌面，支持自定义布局 |
| DataScope | 数据范围策略，控制查询数据边界 |
| DynamicConfig | 动态配置，支持运行时修改 |
| Role | 用户组/角色，作为多态作用主体之一 |
| Subject | 作用主体（User/Role/Team 等），通过多态关联持有模型访问权限 |

## 六、核心类说明

### 6.1 NexusServiceProvider

核心服务提供者，负责：
- 注册模型观察者
- 注入全局 DataScope
- 加载动态配置（`mergePersistentConfig()` 自动合并数据库配置）
- 发布配置文件和语言文件
- 将本包 User 模型注入为 Laravel 默认认证用户模型（`registerUserProvider()`）
- 注册 Builder Macro（`setSubject`/`getSubject`/`hasSubject`）
- 自动检测包变更（`detectPackageChanges()`）
- 注册 Route::auth() 和 Route::tag() 路由宏（`registerRouteMacros()`）

### 6.2 PermissionSyncer

功能权限同步器，负责：
- 解析模块 composer.json 的 permissions 配置
- 落库 permissions 表（不再同步 menus）
- 子节点 tag 自动拼接为 `父tag:子tag`
- 直接通过 package_id 关联包（去掉多态关联）

### 6.3 MenuTreeBuilder

菜单树构建器，负责：
- 从 menus 表构建树形结构
- 动态拼接 path
- 合并 desktop_menus 的 custom 覆盖值

### 6.4 HasDataScope

ORM Trait，负责：
- 通过作用主体（Subject）的 `getModelAccess()` 方法消费模型访问权限
- 自动注入数据范围查询条件
- 读写删权限熔断（抛出 `PermissionDeniedException`）
- 支持 `setScopeSubject()` 注入其他作用主体
- 支持 `withoutDataScope()` 跳过作用域

### 6.5 VerifyAuthTagMiddleware

功能权限标记校验中间件，负责：
- 校验当前用户是否拥有指定功能权限 tag
- **三层降级推断策略**：中间件参数 → defaults('auth_tag') → 自动推断（控制器名:方法映射）
- 检查的是 permissions 表中的功能标记，与 model_accesses 无关
- package_id 安全查询（有 package_id 精确查，无则查全局）

### 6.6 HasModelAccess（契约接口）

模型访问权限主体契约，定义 `getModelAccess(?string $modelClass): Collection` 方法。
User、Role 等模型实现此接口，作为 model_accesses 的多态作用主体。

### 6.7 HasModelAccessTrait

模型访问权限主体通用 Trait，提供：
- `modelAccesses()` → `morphMany(ModelAccess::class, 'subject')` 多态关联
- `getModelAccess()` 默认实现（仅查自身），带缓存支持

### 6.8 User 模型

- 继承 `Illuminate\Foundation\Auth\User`（Authenticatable）
- 实现 `HasModelAccess` 接口
- `getModelAccess()` 穿透逻辑：自身权限 + 角色组权限，用户权限覆盖角色权限
- 通过 `NexusServiceProvider` 注入为 Laravel 默认认证用户模型

### 6.9 Role 模型

- 继承 `Illuminate\Database\Eloquent\Model`
- 实现 `HasModelAccess` 接口（使用 Trait 默认实现，仅查自身）
- 作为多态作用主体之一

### 6.10 PermissionDeniedException

- 继承 `Symfony\Component\HttpKernel\Exception\HttpException`（403）
- 消息通过 `nexus::exceptions.{key}` 语言包加载，支持国际化
- 所有权限拒绝场景统一抛出此异常

### 6.11 Filterable Trait

统一查询过滤 Trait，负责：
- 提供 `scopeFilter()` 作用域，接收结构化过滤条件
- 支持 13 种操作符：eq / neq / like / gt / gte / lt / lte / in / not_in / between / not_between / null / not_null
- 纯查询构建，不参与权限控制
- 配合 FilterRequest 使用

### 6.12 FilterRequest（抽象基类）

统一查询过滤请求基类，负责：
- 子类通过 `filterRules()` 声明式定义"能查什么字段、用什么操作符、默认值是什么"
- `filters()` 将请求参数解析为 Filterable 需要的结构化格式
- 支持默认值（请求未传参时自动应用）
- 自动根据 filterRules 生成 Laravel 验证规则
- `authorize()` 控制谁能访问

### 6.13 TypedValueCast

Laravel Custom Cast，负责：
- 根据 Config 模型的 type 字段动态转换 value 类型
- 支持 boolean / number / json / null / string
- 写入时自动推断类型并序列化

### 6.14 DynamicConfigManager

动态配置管理器，负责：
- 动态配置读写，支持运行时修改
- `mergeIntoConfig()` 将数据库配置合并到 Laravel Config Repository
- `set()` 写入后自动 `config()->set()` 即时生效
- `delete()` 删除后恢复静态配置值

## 七、权限体系

### 7.1 功能权限标记（Permission）

来自模块 composer.json 的 `extra.nexus.permissions` 配置，通过 tag 标识。
同一个 tag 可同时控制前端路由、后端路由、页面元素，实现"一次勾选，前后端都生效"。

权限与路由自动关联：
- `Route::auth()` 自动注入 package_id
- `VerifyAuthTagMiddleware` 三层降级推断 tag
- 支持自动推断、->tag()、原生中间件三种方式

### 7.2 模型访问权限（ModelAccess）

```
User (extends Authenticatable, implements HasModelAccess, use HasModelAccessTrait)
 ├── modelAccesses() → morphMany(ModelAccess::class, 'subject')
 ├── getModelAccess() → 自身权限 + 穿透 roles
 │     ├── $this->modelAccesses() 查自身
 │     └── 遍历 $this->roles → role->getModelAccess()
 └── roles() → BelongsToMany('role_user')

Role (extends Model, implements HasModelAccess, use HasModelAccessTrait)
 ├── modelAccesses() → morphMany(ModelAccess::class, 'subject')
 ├── getModelAccess() → 只查自身（Trait 默认实现）
 └── users() → BelongsToMany('role_user')

ModelAccess
 └── subject() → morphTo() 多态关联作用主体
```

## 九、第三方模块集成流程

```
模块安装
    ↓
composer.json extra.nexus 被扫描
    ↓
PermissionSyncer 解析配置
    ↓
落库 permissions 表（tag 自动拼接、package_id 关联）
    ↓
模块路由使用 Route::auth() 自动注入 package_id
    ↓
中间件自动推断 tag → 校验权限
    ↓
模块卸载时清理权限数据（where('package_id')->delete）
```

## 十、路由权限系统

### 10.1 Route::auth() 宏

自动注入 package_id 到路由 defaults，支持自动推断和显式指定包名。

```php
// 自动推断（推荐，在模块 ServiceProvider 中使用）
Route::auth(function () {
    Route::get('/articles', [ArticleController::class, 'index']);
});

// 显式指定包名
Route::auth('third-party/module-article', function () {
    Route::get('/articles', [ArticleController::class, 'index']);
});
```

### 10.2 Route::tag() 宏

将自定义 tag 写入 defaults('auth_tag')，用于非标准路由。

```php
Route::auth(function () {
    Route::get('/custom', [CustomController::class, 'index'])->tag('custom:tag');
});
```

### 10.3 中间件三层降级

| 方式 | tag 来源 | package_id | 适用场景 |
|------|---------|-----------|---------|
| 自动推断 | 控制器名:方法映射 | 自动注入 | 标准 RESTful 路由 |
| ->tag() | defaults('auth_tag') | 自动注入 | 非标准路由 |
| 原生中间件 | 中间件参数 | null（查全局） | Route::auth() 外部 |

## 十一、设计原则

1. **零物理外键** - 逻辑关联，表结构独立
2. **Laravel 生态兼容** - 遵循命名规范，使用生态工具，支持路由缓存
3. **配置驱动** - 静态配置放 config，动态配置放数据库
4. **模块化** - 第三方模块即装即用
5. **单一职责** - 每个类只做一件事
6. **多态作用主体** - User/Role/Team 等通过多态关联持有模型访问权限
7. **国际化异常** - 权限拒绝异常通过语言包支持多语言
8. **权限概念分离** - Permission（功能标记）与 ModelAccess（模型访问权限）严格区分
9. **声明式过滤** - FilterRequest 定义规则，Filterable 执行查询，Model 不参与
10. **零配置路由权限** - tag 从控制器和方法自动推断，无需手动声明
11. **权限安全** - package_id + tag 复合唯一，杜绝跨包误判
