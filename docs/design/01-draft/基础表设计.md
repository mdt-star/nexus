## 数据库物理 Schema 设计
逻辑强控，无物理外键
迁移生成以下核心业务表：

---

### 2.1 功能权限标记字典表 `permissions`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| parent_id | bigint, unsigned, nullable | 父级 id，null 为顶级 |
| package_id | bigint, unsigned, nullable | 关联 packages.id，null 为全局权限 |
| tag | string, index | 权限标识（如 `article:list`, `article:add`）|
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

唯一约束：`unique(['package_id', 'tag'])`，不同包可以有同名 tag。

> name 通过多国语言包匹配 tag 获取，不存储在数据库。
> 本表存储的是功能权限标记（Feature Flag），与 model_accesses（模型访问权限）无关。

---

### 2.2 菜单发布池 `menus`

通过 parent_id 与 permissions 表保持一致的树形层级关系。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| parent_id | bigint, unsigned, nullable | 父级 id，null 为顶级（关联 permissions.id）|
| label | string | 菜单标题 |
| path | string | 路由路径（相对路径，动态拼接时使用）|
| component | string, nullable | 前端组件路径（如 `Article/List`）|
| icon | string, nullable | 图标类名 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> 落库时 path 保持原始值，数据返回时动态拼接完整路径。

---

### 2.3 用户桌面配置表 `desktops`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| user_id | bigint, unsigned, index | 用户 id |
| name | string | 桌面名称 |
| region | string, index | 布局区域（如 `sidebar_left`, `header`, `main`）|
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.4 桌面菜单关联表 `desktop_menus`

用户可将菜单从 menus 拖拽到桌面，并支持自定义覆盖原菜单项的值。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| desktop_id | bigint, unsigned, index | 关联 desktops.id |
| menu_id | bigint, unsigned, index | 关联 menus.id |
| custom | json, nullable | 自定义覆盖值（如 `{"label": "我的文章", "path": "my-article"}`）|
| sort | int | 排序 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> 模型层自动合并 custom 与 menus 原值，优先使用 custom。

---

### 2.5 分布式动态配置持久化表 `configs`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| key | string, unique, index | 完整配置键名（必须带前缀，如 `core.upload_max_size`）|
| value | text | 持久化配置值（数组存储为 JSON 字符串）|
| type | string, default 'string' | 类型自省标签（`string`, `boolean`, `json`, `number`, `null`）|
| description | string, nullable | 配置说明 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> value 字段通过 TypedValueCast 根据 type 字段自动转换 PHP 类型。

---

### 2.6 数据范围策略字典表 `model_scopes`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| key | string, unique, index | 隔离策略唯一 Key（如 `article_editor_group`）|
| class | string, unique | 对应后端策略类的全限定命名空间（FQCN）|
| model_whitelist | json, nullable | 该策略原生声明的模型白名单数组，null 表示不限制模型 |
| fields_whitelist | json, default '["*"]' | 该策略原生声明的物理字段白名单数组，默认 ["*"] 表示允许所有字段 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.7 模型访问权限表 `model_accesses`

控制主体（用户/角色/团队等）对特定模型的读写删权限及数据范围。
与 permissions（功能标记表）不同，本表控制的是数据层面的访问权限。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| subject_type | string, index | 多态关联类型（作用主体模型全限定类名，如 `App\Models\User`）|
| subject_id | bigint, unsigned, index | 多态关联 ID |
| class | string, index | 强控的 Eloquent Model 全限定名（FQCN）|
| can_read | boolean, default 1 | 是否允许读取 |
| can_write | boolean, default 1 | 是否允许写入/更新 |
| can_delete | boolean, default 1 | 是否允许删除 |
| scope_key | string, nullable, index | 绑定的 Scope Key |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> subject_type + subject_id 为多态关联，支持 user/role/team 等任意作用主体类型。

---

### 2.8 用户组/角色表 `roles`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| name | string | 角色名称 |
| slug | string, unique, index | 角色标识（如 `admin`, `editor`）|
| description | string, nullable | 角色描述 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> Role 作为多态作用主体之一，通过 model_accesses.subject 多态关联持有模型访问权限。

### 表关系概览

```
permissions ──── package_id → packages
                │
menus ───────────────────────────
                                 │
desktops ──────── desktop_menus ──┘
                                 │
model_accesses ──────────────────┘
      │
      ├── subject (morphTo) → User / Role / Team ...
      │
model_scopes ────────────────────┘
                                 │
packages ────────────────────────┘
      │
      └── permissions (HasMany)

configs

users ─── role_user ─── roles
                          │
                          └── model_accesses (作为多态作用主体)
```

### 命名约定

| 旧名称 | 新名称 | 说明 |
|--------|--------|------|
| pid | parent_id | Laravel 生态惯例 |
| position | region | 布局区域定义 |
| routes | menus | 避免与后端路由冲突 |
| data_scopes | model_scopes | 语义更明确 |
| model_permissions | model_accesses | 避免与 permissions（功能标记）混淆 |
