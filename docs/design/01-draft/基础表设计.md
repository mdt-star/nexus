## 数据库物理 Schema 设计
逻辑强控，无物理外键
迁移生成以下核心业务表：

---

### 2.1 功能权限节点字典表 `permissions`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| parent_id | bigint, unsigned, nullable | 父级 id，null 为顶级 |
| tag | string, unique, index | 唯一标识契约（如 `article`, `article:add`）|
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> name 通过多国语言包匹配 tag 获取，不存储在数据库。

---

### 2.2 菜单发布池 `menus`

通过 parent_id 与 permissions 表保持一致的树形层级关系。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| parent_id | bigint, unsigned, nullable | 父级 id，null 为顶级（关联 permissions.id）|
| label | string | 菜单标题 |
| path | string | 路由路径（相对路径，动态拼接时使用）|
| component | string, nullable | 前端组件路径（如 `Article/List`）|
| icon | string, nullable | 图标类名 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> 落库时 path 保持原始值，数据返回时动态拼接完整路径。

---

### 2.3 用户桌面配置表 `desktops`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| user_id | bigint, unsigned, index | 用户 id |
| name | string | 桌面名称 |
| region | string, index | 布局区域（如 `sidebar_left`, `header`, `main`）|
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.4 桌面菜单关联表 `desktop_menus`

用户可将菜单从 menus 拖拽到桌面，并支持自定义覆盖原菜单项的值。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| desktop_id | bigint, unsigned, index | 关联 desktops.id |
| menu_id | bigint, unsigned, index | 关联 menus.id |
| custom | json, nullable | 自定义覆盖值（如 `{"label": "我的文章", "path": "my-article"}`）|
| sort | int | 排序 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

> 模型层自动合并 custom 与 menus 原值，优先使用 custom。

---

### 2.5 分布式动态配置持久化表 `configs`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| config_key | string, unique, index | 完整配置键名（必须带前缀，如 `core.upload_max_size`）|
| config_value | text | 持久化配置值（数组存储为 JSON 字符串）|
| value_type | string | 类型自省标签（`string`, `boolean`, `json`, `number`）|
| description | string, nullable | 配置说明 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.6 数据范围策略字典表 `model_scopes`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| scope_key | string, unique, index | 隔离策略唯一 Key（如 `article_editor_group`）|
| scope_name | string | 前端看到的友好名称 |
| class_path | string, unique | 对应后端策略类的全限定命名空间（FQCN）|
| model_whitelist | json | 该策略原生声明的模型白名单数组 |
| fields_whitelist | json | 该策略原生声明的物理字段白名单数组 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.7 模型多维控制矩阵权限表 `model_permissions`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| subject_type | string, index | 授权主体类型：`user` 或 `role` |
| subject_id | bigint, unsigned, index | 对应主体表的主键 ID |
| model_class | string, index | 强控的 Eloquent Model 全限定名（FQCN）|
| can_read | boolean, default 1 | 是否允许读取 |
| can_write | boolean, default 1 | 是否允许写入/更新 |
| can_delete | boolean, default 1 | 是否允许删除 |
| scope_key | string, nullable, index | 绑定的 Scope Key（对应 `model_scopes.scope_key`）|
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.8 包管理基础表 `packages`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| name | string, unique | Composer 包名（如 `your-project/core-base-module`）|
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2.9 多态归属关联中介表 `package_relations`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint, unsigned, PK, auto_increment | 主键 |
| package_id | bigint, unsigned, index | 关联 packages.id |
| relationable_type | string, index | 关联的目标模型类名（如 `Permission::class`）|
| relationable_id | bigint, unsigned, index | 关联的目标模型主键 ID |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 表关系概览

```
permissions ─────────────┐
                         │
menus ───────────────────┤
                         │
desktops ────────────────┼── desktop_menus ── menus
                         │
model_permissions ───────┘
                         │
model_scopes ────────────┘
                         │
packages ────────────────┼── package_relations ── (permissions, menus, ...)
                         │
configs ────────────────┘
```

### 命名约定

| 旧名称 | 新名称 | 说明 |
|--------|--------|------|
| pid | parent_id | Laravel 生态惯例 |
| position | region | 布局区域定义 |
| routes | menus | 避免与后端路由冲突 |
| data_scopes | model_scopes | 语义更明确 |
