# 项目架构设计

## 一、项目概述

`core-base-module` 是一个 Laravel 核心权限模块，提供功能权限控制、菜单管理、数据范围策略、动态配置等基础设施能力。

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| Permission | 功能权限节点，通过 tag 标识 |
| Menu | 菜单发布池，前端可用的菜单项 |
| Desktop | 用户桌面，支持自定义布局 |
| DataScope | 数据范围策略，控制查询数据边界 |
| DynamicConfig | 动态配置，支持运行时修改 |

## 五、文件命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 模型 | 单数，PascalCase | `Permission.php` |
| 控制器 | PascalCase + Controller | `PermissionController.php` |
| 中间件 | PascalCase + Middleware | `VerifyAuthTagMiddleware.php` |
| 服务类 | PascalCase + Manager/Service | `DynamicConfigManager.php` |
| Trait | PascalCase | `HasDataScope.php` |
| 接口 | PascalCase + Interface | `DataScopeStrategyInterface.php` |
| 命令 | PascalCase + Command | `SyncPermissionsCommand.php` |
| 迁移 | `create_{table}_table.php` | `create_permissions_table.php` |
| 配置 | snake_case | `platform_core.php` |

## 六、核心类说明

### 6.1 CoreServiceProvider

核心服务提供者，负责：
- 注册模型观察者
- 注入全局 DataScope
- 加载动态配置
- 发布配置文件和语言文件

### 6.2 PermissionSyncer

权限同步器，负责：
- 解析模块 composer.json 的 permissions 配置
- 落库 permissions 和 menus 表
- 保持树形结构

### 6.3 MenuTreeBuilder

菜单树构建器，负责：
- 从 menus 表构建树形结构
- 动态拼接 path
- 合并 desktop_menus 的 custom 覆盖值

### 6.4 HasDataScope

ORM Trait，负责：
- 自动注入数据范围条件
- 处理字段别名映射
- 读写删权限熔断

### 6.5 VerifyAuthTagMiddleware

权限校验中间件，负责：
- 校验当前用户的 auth_tag
- 返回 403 或重定向

## 八、第三方模块集成流程

```
模块安装
    ↓
composer.json extra.platform-module 被扫描
    ↓
PermissionSyncer 解析配置
    ↓
落库 permissions + menus 表
    ↓
模块卸载时清理关联数据
```

## 九、设计原则

1. **零物理外键** - 逻辑关联，表结构独立
2. **Laravel 生态兼容** - 遵循命名规范，使用生态工具
3. **配置驱动** - 静态配置放 config，动态配置放数据库
4. **模块化** - 第三方模块即装即用
5. **单一职责** - 每个类只做一件事
